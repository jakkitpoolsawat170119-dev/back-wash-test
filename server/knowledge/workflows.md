# ขั้นตอนการทำงาน (Workflows)

## สายการผลิต
- **Line 1** = Syrup · **Line 2, Line 3** = Flavour · **Line 4** = Mixing/Pasteurizer (บันทึกแบบ manual + รายงานบรรจุ)

## การบันทึกผลิต
ปกติพนักงานกด Start → Done ที่หน้า Production Control ทีละ batch ระบบบันทึก production_logs + แจ้ง Telegram + ส่งเข้า Google Sheet ถ้าบันทึกผ่านผู้ช่วย AI ใช้ tool record_production (ต้องให้ผู้ใช้กดยืนยันก่อน) ค่า Brix/pH ใส่หรือไม่ใส่ก็ได้

## การทำ CIP
- CIP ต้องทำเมื่อเปลี่ยนรสชาติ หรือครบรอบเวลาที่กำหนด
- Line 1 ใช้ฟอร์ม CIP Line 1 · Line 2/3 ใช้ฟอร์ม CIP Line 2/3 (มีตัวเลือก Backwash ต่อรอบ)
- 1 session = การเปิดฟอร์ม 1 ครั้ง มีได้หลายรอบ (row) รอบที่จบแล้วมี endTime
- ถ้าบันทึกผ่านผู้ช่วย AI ใช้ tool record_cip_round (ต้องยืนยันก่อน) — เหมาะกับบันทึกย้อนหลัง/แก้ตกหล่น ถ้าจะกรอกรายละเอียดครบให้ไปที่หน้าฟอร์ม CIP โดยตรง

## แผนผลิต + To-do
วางแผนที่ปุ่ม "📋 แผนผลิตวันนี้" ในหน้าผลิต หรือผ่าน AI (tool update_production_plan ต้องยืนยัน) เมื่อบันทึกแผน ระบบสร้างงาน To-do อัตโนมัติ (source='auto_plan') และติ๊กความคืบหน้าให้เองจากยอดผลิตจริง

## ส่งเวร / รับกะ
สิ้นกะกรอกฟอร์มส่งกะ (สถานะราย Line + Lot No.) → บันทึก handover_notes + แจ้ง Telegram กะใหม่กดรับกะ ผ่าน AI ใช้ save_handover_note (โน้ตข้อความ ต้องยืนยัน)

## รายงานและแจ้งเตือน
- ทุกการบันทึกสำคัญแจ้งเข้า Telegram กลุ่มโรงงานอัตโนมัติ
- ถามกราฟผลิตใน Telegram: พิมพ์ "สรุปผลิต", "สรุปผลิต Amazon", "สรุปผลิต Amazon 2026-06-11", "สรุปผลิต ทั้งหมด"
- หน้า To-do แท็บ "ส่งรายงาน" ตั้งเวลาส่งรายงานอัตโนมัติได้ (ตามสิ้นกะหรือเวลาที่กำหนด)

## หลักการตอบของผู้ช่วย AI
- ไม่รู้ → ค้น search_knowledge / query_database ก่อน ห้ามเดา ถ้าหาไม่เจอให้บอกตรงๆ ว่าไม่พบข้อมูล
- การเขียนข้อมูล (ผลิต/CIP/แผน/ส่งเวร) ทุกครั้งต้องผ่านการยืนยันจากผู้ใช้ก่อน ระบบจะขึ้นการ์ดให้กด ✅ ยืนยัน
